# Environment Variables Setup Guide

This guide will help you fill in the `.env` file with the correct values.

## 🚀 Quick Start

1. The `.env` file has been created in the root folder
2. Follow the sections below to get each required value
3. Copy the same values to GitHub Secrets for deployment

## 📋 Step-by-Step Setup

### 1️⃣ AWS Credentials

**Get your AWS Access Keys:**

```bash
# Option 1: Using AWS CLI
aws iam create-user --user-name github-actions-salesai
aws iam attach-user-policy \
  --user-name github-actions-salesai \
  --policy-arn arn:aws:iam::aws:policy/AdministratorAccess
aws iam create-access-key --user-name github-actions-salesai
```

**Or use AWS Console:**
- Go to IAM → Users → Create user → Security credentials → Create access key
- Save the Access Key ID and Secret Access Key

**Update in `.env`:**
```bash
AWS_ACCESS_KEY_ID=AKIAIOSFODNN7EXAMPLE
AWS_SECRET_ACCESS_KEY=wJalrXUtnFEMI/K7MDENG/bPxRfiCY
AWS_REGION=us-east-1
```

### 2️⃣ Database Password

Choose a strong password for your PostgreSQL database:

```bash
DB_PASSWORD=MySecurePassword123!@#
```

**Requirements:**
- At least 12 characters
- Mix of uppercase, lowercase, numbers, special characters
- No common words or patterns

### 3️⃣ Clerk Authentication

**Get Clerk keys:**

1. Go to [clerk.com](https://clerk.com) and sign up
2. Create a new application
3. Go to **API Keys** in the dashboard
4. Copy the values:

```bash
CLERK_DOMAIN=your-app-name.clerk.accounts.dev
CLERK_SECRET_KEY=sk_test_xxxxxxxxxxxxxxxxxxxxxxxxxx
CLERK_PUBLISHABLE_KEY=pk_test_xxxxxxxxxxxxxxxxxxxxxxxxxx
```

**Also update:**
```bash
VITE_CLERK_PUBLISHABLE_KEY=pk_test_xxxxxxxxxxxxxxxxxxxxxxxxxx
TF_VAR_clerk_publishable_key=pk_test_xxxxxxxxxxxxxxxxxxxxxxxxxx
```

### 4️⃣ OpenAI API Key

**Get OpenAI key:**

1. Go to [platform.openai.com](https://platform.openai.com)
2. Sign in / Sign up
3. Go to **API Keys** section
4. Click **Create new secret key**
5. Copy the key (starts with `sk-proj-` or `sk-`)

```bash
OPENAI_API_KEY=sk-proj-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
TF_VAR_openai_api_key=sk-proj-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### 5️⃣ Telegram Bot Token

**Create a Telegram bot:**

1. Open Telegram and search for `@BotFather`
2. Send `/newbot` command
3. Follow the prompts:
   - Choose a name for your bot (e.g., "My Sales Helper")
   - Choose a username (must end in "bot", e.g., "mysales_helper_bot")
4. Copy the token provided

```bash
TELEGRAM_BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz
TF_VAR_telegram_bot_token=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz
```

### 6️⃣ Deploy Infrastructure (First Time)

After setting the above 5 values, deploy to AWS:

**Option A: Using GitHub Actions**
1. Copy all values to GitHub Secrets
2. Push to main branch: `git push origin main`

**Option B: Using Terraform locally**
```bash
cd terraform
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars with your values
terraform init
terraform apply
```

### 7️⃣ Update Post-Deployment Values

After infrastructure deploys, update these in `.env`:

**Get values from Terraform output:**
```bash
cd terraform
terraform output
```

**Or from AWS Console:**
- API Gateway URL: AWS Console → API Gateway → Stages → Prod
- S3 Bucket: AWS Console → S3 → Buckets
- CloudFront: AWS Console → CloudFront → Distributions

**Update in `.env`:**
```bash
API_URL=https://abc123xyz.execute-api.us-east-1.amazonaws.com/api
VITE_API_URL=https://abc123xyz.execute-api.us-east-1.amazonaws.com/api

S3_FRONTEND_BUCKET=salesai-frontend-1234567890
S3_BUCKET_IMAGES=salesai-product-images-1234567890

CLOUDFRONT_DISTRIBUTION_ID=E1234567890ABC
CLOUDFRONT_DOMAIN=d1234567890.cloudfront.net

TELEGRAM_WEBHOOK_URL=https://abc123xyz.execute-api.us-east-1.amazonaws.com/api/telegram/webhook
```

### 8️⃣ Add to GitHub Secrets

Go to your GitHub repo → **Settings** → **Secrets and variables** → **Actions**

Add each of these as a new repository secret:

**Required before first deploy:**
- `AWS_ACCESS_KEY_ID`
- `AWS_SECRET_ACCESS_KEY`
- `DB_PASSWORD`
- `OPENAI_API_KEY`
- `TELEGRAM_BOT_TOKEN`
- `CLERK_DOMAIN`
- `CLERK_SECRET_KEY`
- `CLERK_PUBLISHABLE_KEY`

**Required after first deploy:**
- `API_URL`
- `S3_FRONTEND_BUCKET`
- `CLOUDFRONT_DISTRIBUTION_ID`
- `CLOUDFRONT_DOMAIN`

## ✅ Verification Checklist

Before deploying, verify:

- [ ] AWS credentials are valid (test with `aws sts get-caller-identity`)
- [ ] Database password is strong and saved
- [ ] Clerk application is created and keys copied
- [ ] OpenAI API key is valid and has credits
- [ ] Telegram bot is created and token copied
- [ ] All GitHub Secrets are added
- [ ] `.env` file is NOT committed to git (check `.gitignore`)

## 🔒 Security Best Practices

1. **Never commit `.env` to git** - It's already in `.gitignore`
2. **Rotate keys regularly** - Every 90 days for AWS
3. **Use different keys for dev/prod** - Clerk has test and live keys
4. **Monitor usage** - Check AWS billing, OpenAI usage
5. **Delete old keys** - If you create new ones, delete the old

## 🆘 Troubleshooting

### "AWS credentials not found"
```bash
# Test your credentials
aws sts get-caller-identity

# If fails, reconfigure
aws configure
```

### "Clerk keys invalid"
- Make sure you're using keys from the correct application
- Check if you're using test keys vs live keys
- Verify domain matches (includes .clerk.accounts.dev)

### "OpenAI API key invalid"
- Check if key starts with `sk-`
- Verify account has credits
- Make sure key hasn't expired or been revoked

### "Terraform fails"
- Ensure all TF_VAR_ variables are set
- Check AWS credentials have proper permissions
- Verify region has capacity (try different region if needed)
